Partial Randomization

In simulations, sometimes we need to explore the graphs with certain restrictions; we need to fix some (not all) parameters that models our restrictions, while randomizing the rest of the parameters. As an example, for testing an ML or causal inference model with Gaussian assumption on the nodes, we want to simulate a graph with all Gaussian output distributions, while allowing the mean values to be selected randomly (because perhaps we want to test our model on MVG distributions with arbitrary means and variances).

An important feature of PARCS, the partial randomization, allows us to simulate for such scenarios. In this section we explain how this can be done.

Parameter Randomization

As before, simulation starts with a graph description file. However we can use two new values to activate randomization PARCS: ? for parameters, and random for functions and distributions. Here is an example of the causal triangle where we randomize the mean of node A, and also the output distribution of Y:

graph_description.yml
1# nodes
2C: gaussian(mu_=1, sigma_=1)
3A: gaussian(mu_=?, sigma_=1)
4Y: random
5# edges
6C->A: identity()
7C->Y: identity()
8A->Y: identity()
graph.py
 1from pyparcs.cdag.graph_objects import Graph
 2from pyparcs.graph_builder.parsers import graph_file_parser
 3from pyparcs.graph_builder.randomizer import ParamRandomizer
 4
 5rndz = ParamRandomizer(
 6    graph_dir='graph_description_1.yml',
 7    guideline_dir='simple_guideline.yml'
 8)
 9nodes, edges = rndz.get_graph_params()
10
11g = Graph(nodes=nodes, edges=edges)
12samples = g.sample(size=3)
13print(samples)
14#           C         A    Y
15# 0  1.660388  0.410814  1.0
16# 1  1.253973 -2.983480  0.0
17# 2  1.088486 -0.167692  1.0

In graph.py file, instead of graph_file_parser, we import parcs.graph_builder.randomizer.ParamRandomizer. ParamRandomizer takes the graph description directory, as well as a guideline file. This file tells the radomizer, what are the options for parameters and functions. Let’s have a look at the guideline file:

simple_guideline.yml
 1nodes:
 2  bernoulli:
 3    p_: [ [f-range, 1, 2] , 0 , [f-range, 2, 3] ]
 4  gaussian:
 5    mu_: [ [i-range, -2, -1] , [f-range, 0.5, 1] , 0 ]
 6    sigma_: [ [f-range, 1, 3] , 0 , 0 ]
 7edges:
 8  identity: null
 9  gaussian_rbf:
10    alpha: [f-range, 1, 4]
11    beta: [f-range, -0.5, 0.5]
12    gamma: [choice, 0, 2]
13    tau: 2

  1. There are two main keys: nodes and edges

  2. the next-level keys determine the different possibilities for choosing the functions and distributions. For instance, in the example file we tell the randomizer to choose between bernoulli and gaussian for distributions, and between identity and gaussian_rbf for the edge functions.

  3. Each function/distribution option contains the corresponding parameters as keys. The value of these keys is a list of the length 3 which corresponds to bias, linear, and interactions coefficients (in the same order). Each component is a directive for the respective coefficient, i.e. first component is used to randomize the bias value, second one for linear coefficient, and third for the interaction coefficient. The directives follow the following convention:

    1. a single value, means fixed, thus for all the randomization turns, randomizer picks the same value

    2. [f-range, X, Y] tells the randomizer to pick a float value between X and Y uniformly, i.e. from Unif(X, Y).

    3. [i-range, X, Y] is similar to (a), except that the value is integer (with equal probabilities).

    4. [choice, X_0, X_1, ...] tells the randomizer to pick a value from the given list of options in the directive with equal probabilities (element 2nd onward).

In our example, therefore, mean of node A is selected according to line 5 of the guideline, and the distribution of Y is chosen between Bernoulli and Gaussian distributions.

Note

For f-range and i-range directives, we can specify more than one range. To do so, we simply write the next low/high pairs in the list. Example: The guideline [f-range, -1, -0.5, 0.5, 1] samples randomly from the range of \((-1, -0.5) \cup (0.5, 1)\).

Randomization conventions

Free Randomizer

Using free randomizer, we can randomly generate a graph based on guidelines for number of nodes and graph sparsity. In fact, this randomizer is wrapped around the parameter randomization, meaning that first, PARCS initializes a graph with some nodes and edges, and assigns free and ? to all functions and parameters, then passes the process to parameter randomizer to fill the blanks.

The guideline is similar to that of parameter randomizer, except that we need to give the directives for graph as well. The keys are num_nodes and graph_sparsity.

simple_guideline.yml
 1graph:
 2  num_nodes: [i-range, 2, 5]
 3  graph_density: [f-range, 0.8, 1]
 4nodes:
 5  bernoulli:
 6    p_: [ [f-range, 1, 2] , 0 , [f-range, 2, 3] ]
 7  gaussian:
 8    mu_: [ [i-range, -2, 2] , [f-range, -2, 2] , 0 ]
 9    sigma_: [ [f-range, 1, 3] , 0 , 0 ]
10edges:
11  identity: null
12  gaussian_rbf:
13    alpha: [f-range, 1, 4]
14    beta: [f-range, -0.5, 0.5]
15    gamma: [choice, 0, 2]
16    tau: 2

using the FreeRandomizer class, we generate a graph object.

graph.py
 1from pyparcs.cdag.graph_objects import Graph
 2from pyparcs.graph_builder.randomizer import FreeRandomizer
 3
 4rndz = FreeRandomizer(
 5    guideline_dir='simple_guideline.yml'
 6)
 7nodes, edges = rndz.get_graph_params()
 8
 9g = Graph(nodes=nodes, edges=edges)
10samples = g.sample(size=3)
11print(samples)
12#         H_0  H_1       H_2       H_3
13# 0  0.641136  1.0 -2.300603  0.343558
14# 1 -2.007259  1.0  3.145062  1.988338
15# 2  0.874383  1.0 -3.203189  1.714673

Nodes are named H_<num> by default. You can change the name prefix by the key node_name_prefix in the guideline (under graph key). The value must be string.

Connect Randomizer

This class randomizes the causal flow from a parent to a child graph. To clarify this functionality, suppose we want to simulate the following data model.

\[\begin{split}\begin{aligned} L &= AL \\ Z &= BZ + CL \end{aligned}\end{split}\]

where \(Z\) and \(L\) are random vectors, \(A\) and \(B\) are a lower triangular coefficient matrices, and \(C\) is another arbitrary coefficient matrix. In this model, \(L\) and \(Z\) are normal simulated graphs, while \(Z\) receives causal edges from \(L\) via \(C\) matrix.

This model can be simulated as the following:

graph description for L
1# === A causal Triangle: Treatment, Outcome, Confounder ===
2# nodes
3L_1: gaussian(mu_=0, sigma_=1)
4L_2: gaussian(mu_=2L_1-1, sigma_=1)
5L_3: gaussian(mu_=L_2+L_1, sigma_=1)
6# edges
7L_1->L_2: identity()
8L_1->L_3: identity()
9L_2->L_3: identity()

This description is a normal PARCS graph description file for the parent graph.

graph description for Z
1# === A causal Triangle: Treatment, Outcome, Confounder ===
2# nodes
3Z_1: gaussian(mu_=0, sigma_=1)
4Z_2: gaussian(*mu_=2Z_1, sigma_=1)
5Z_3: gaussian(*mu_=Z_1+Z_2, sigma_=1)
6# edges
7Z_1->Z_2: identity()
8Z_1->Z_3: identity()
9Z_2->Z_3: identity()

The second graph description file has a minor special style. In line 4 and 5 you can see a symbol * behind mu_ distribution parameters. This syntax tells PARCS to affect these parameters when external causal edges (from L) connect to the graph. Similarly, the fact that the node Z_1 doesn’t have an asterisk means that PARCS will not let any edge to go to Z_1.

Similar to other randomization classes, ConnectRandomizer also requires a (normal) guideline file.

Finally, the graph can be simulated as follows:

graph.py
 1from pyparcs.cdag.graph_objects import Graph
 2from pyparcs.graph_builder.randomizer import ConnectRandomizer
 3import pandas as pd
 4import numpy as np
 5np.random.seed(2022)
 6
 7
 8rndz = ConnectRandomizer(
 9    parent_graph_dir='graph_description_L.yml',
10    child_graph_dir='graph_description_Z.yml',
11    guideline_dir='simple_guideline.yml',
12    adj_matrix_mask=pd.DataFrame(np.ones(shape=(3, 3)),
13                                 index=('L_1', 'L_2', 'L_3'),
14                                 columns=('Z_1', 'Z_2', 'Z_3'))
15)
16nodes, edges = rndz.get_graph_params()
17g = Graph(nodes=nodes, edges=edges)
18print(g.sample(4))
19#         L_1       Z_1       L_2       L_3        Z_2        Z_3
20# 0  0.192125 -0.523060 -1.064670 -1.345416  -4.565572  -5.873356
21# 1 -1.383525  0.079110 -3.010643 -5.204452 -11.578450 -12.299672
22# 2  1.675641 -1.298673  2.850752  4.498497   7.355618   5.460474
23# 3 -0.546389  0.540718 -0.436297 -1.625131  -2.717709  -1.557761

argument adj_matrix_mask is a further restriction by the user to limit the new edges. For example we can suppress any outgoing edge from L_1 if in this mask, first row is all zero (rows and columns follow the same order of the nodes in the graph description file of the parent and child graphs respectively).

ConnectionRandomizer creates a temp graph description file as an intermediate step, which deletes it after the process if delete_temp_graph_description=False. To check how the class works, we can have a look at this temp file.

temp file: combined_gdf.yml
 1L_1: gaussian(mu_=0, sigma_=1)
 2L_2: gaussian(mu_=2L_1-1, sigma_=1)
 3L_3: gaussian(mu_=L_2+L_1, sigma_=1)
 4Z_1: gaussian(mu_=0,sigma_=1)
 5Z_2: gaussian(mu_=2Z_1+1.31L_2+1.33L_3,sigma_=1)
 6Z_3: gaussian(mu_=Z_1+Z_2+1.83L_3,sigma_=1)
 7L_1->L_2: identity()
 8L_1->L_3: identity()
 9L_2->L_3: identity()
10L_2->Z_2: identity()
11L_3->Z_2: identity()
12L_3->Z_3: gaussian_rbf(?)
13Z_1->Z_2: identity()
14Z_1->Z_3: identity()
15Z_2->Z_3: identity()

The highlighted lines show the traces of the new connection between graphs. Note that this description file is next passed to the ParamRandomizer, the parent class of ConnectionRandomizer, and that is why there are question marks in the description file.

Note

The density of new edges follows the graph_density entry in the guideline file. However, based on asterisks and adjacency matrix mask, number of edges might be less than intended. E.g., graph_density: 1 tells PARCS to fully connect the parent graph to child graph. However, if the child nodes doesn’t have asterisks in parameters, PARCS will skip all nodes (as we have requested no parameter to be affected by new edges), hence no new edges will be added.

Edge correction

When randomizing the graph parameters, we potentially face two undesired issues:

  1. For edge functions such as Sigmoid and Gaussian RBF, if the parent output is too far away from the active region of the function, we will end up with low variability in the transformed values. For example, If for a Sigmoid edge function, the output distribution of the parent node is \(\mathcal{N}(10, 2)\), then the transformed values of a Sigmoid edge function for 1000 samples are in the range of \((0.9912\dots, 0.9999\dots)\).

  2. Take an example of a node with 3 parents, 2 of which are subjected to Sigmoid edge function and have a \([0, 1]\) support while calculating the child node’s distribution parameters, while the remaining is subjected to the identity function and retain its original support. In this case, we have an imbalance when calculating the distribution coefficients, as we pick the distribution coefficients randomly from the same value range for all parents.

In general, both items indicate an issue similar to the issue of sensitivity of calculations to unit of measurement, and of course the remedy to that can be the same, which is normalization. In issue 1, normalization brings the values to the active region of the edge function, and in issue 2, it brings all the parents to the same scale.

Such normalization can be done via activating the correction option for the edges. PARCS automatically activates edge correction when randomizing the parameters. But in general, we can do edge correction in any simulation, by modifying the graph description file.

graph_description.yml
1# === A causal Triangle: Treatment, Outcome, Confounder ===
2# nodes
3C: gaussian(mu_=2, sigma_=1)
4A: gaussian(mu_=10, sigma_=1)
5Y: bernoulli(p_=C+A), correction[]
6# edges
7C->A: identity()
8C->Y: identity()
9A->Y: identity()
graph.py
1from pyparcs.cdag.graph_objects import Graph
2from pyparcs.graph_builder.parsers import graph_file_parser
3
4nodes, edges = graph_file_parser('graph_description_edge.yml')
5g = Graph(nodes=nodes, edges=edges)
6
7samples = g.sample(size=1000)
8print(samples['Y'].mean())
9# 1.0

In this example, the variable Y has the mean of 1.0 because of the mean of A and C. Now we activate correction for A->Y and C->Y edges:

graph_description.yml
1# === A causal Triangle: Treatment, Outcome, Confounder ===
2# nodes
3C: gaussian(mu_=2, sigma_=1)
4A: gaussian(mu_=10, sigma_=1)
5Y: bernoulli(p_=C+A), correction[]
6# edges
7C->A: identity()
8C->Y: identity(), correction[]
9A->Y: identity(), correction[]
graph.py
1nodes, edges = graph_file_parser('graph_description_edge_2.yml')
2g = Graph(nodes=nodes, edges=edges)
3
4samples = g.sample(size=1000)
5print(samples['Y'].mean())
6# 0.477

Note

Similar to Node correction edge correction parameters (i.e. mean and standard deviation) is always initialized upon the first batch of data. For this purpose, the graph object always burns the first 500 samples, to initialize the corrections. Read more about edge correction at EdgeCorrection().
